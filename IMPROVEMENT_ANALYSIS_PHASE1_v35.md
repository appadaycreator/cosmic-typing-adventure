## 1️⃣ P1: ランディングページ（index.html）CTA配置の最適化

### 目的
ファーストビュー（モバイル・PC双方）で「今すぐプレイ」ボタンの視認性が低い。ヒーロセクションのメインメッセージの大きさが小さく、訪問者が「このサイトは何か？」を即座に理解しにくい。

### 仕様
1. **ヒーロセクション h1 タグのフォントサイズ拡大**
   - モバイル（< 768px）: `2rem` → `2.5rem` に拡大
   - デスクトップ（≥ 768px）: `3rem` → `3.5rem` に拡大
   - line-height: 1.2 を指定（行間を詰める）

2. **CTA ボタンの視認性向上**
   - 背景色: 現在の`#7c3aed` → `#fbbf24`（ゴールド）に変更
   - テキスト: 「今すぐプレイ」→ 「さっそくプレイ！🚀」に変更
   - Padding: `py-3 px-6` → `py-4 px-8` に拡大（タッチターゲット: 56px以上）
   - Font Weight: bold (既存) に加えて font-size: 1.125rem に拡大

3. **信頼度シグナル（Social Proof）の追加**
   - ヒーロ下部に「✨ 10,000+ の利用者に選ばれています」テキストを追加
   - 小さく「評価 ⭐4.8/5.0」も表示（灰色テキスト）

4. **背景アニメーションの強化**（既存の`cosmic-gradient`を活用）
   - パーティクル点滅アニメーション（既に実装済み）
   - グラデーション方向: 135deg で固定（アニメーション不要）

### 実装タスク
1. `index.html` の hero セクション（`<section class="hero-section ...">` または最初の主要セクション）の h1 タグの Tailwind クラスを編集
   - 既存: `text-3xl md:text-5xl` → `text-2.5xl md:text-3.5xl` に変更（Tailwind にない size の場合、style タグで直接指定）
2. CTA ボタンのクラスを編集
   - `bg-nebula-purple` → `bg-yellow-400` に変更（または `#fbbf24` を style で指定）
   - `text-sm` → `text-lg` に変更
3. Social Proof テキストを h1 の直下に追加（`<p class="text-gray-400 text-sm mt-2">...</p>`）
4. CSS: 必要に応じて カスタム style タグで `h1 { font-size: 2.5rem; line-height: 1.2; }` 等を追加

### DoD
- ✅ ボタンがモバイル（375px 幅）で画面内に収まる
- ✅ ボタンのクリック/タップ可能エリアが 56px × 56px 以上
- ✅ h1 テキストが 2 行以内（モバイル）
- ✅ CTA ボタンの色がゴールド（#fbbf24 または同等）に変更される
- ✅ ローカルサーバ（localhost:8000）で動作確認

## 2️⃣ P2: ゲーム結果表示後の「次へ進む」ナビゲーション改善

### 目的
タイピング完了後、結果パネルが表示されるが「次のミッション選択に進む」ボタンが不明確または配置が悪い。ユーザーが「何をしたらいいのか」を迷い、セッション放棄のリスク。

### 仕様
1. **結果パネル（`#resultPanel`）の下部に「← ミッション選択へ戻る」ボタンを追加**
   - テキスト: 「← ミッション選択へ」（左矢印で戻る意図を明示）
   - 背景色: `#7c3aed`（紫、既存テーマ色）
   - Padding: `py-3 px-6`（56px以上）
   - Font: `text-lg font-bold`
   - 配置: 既存の「📸 カード保存」「共有」ボタンの下部に配置

2. **モバイル時のボタンレイアウト調整**
   - PC（≥ 768px）: 横並び（flex-row）、ボタン間に `gap-2` を指定
   - モバイル（< 768px）: 縦積み（flex-col）で配置（タップ誤操作防止）

3. **クリック処理**
   - onclick: `resetSession()` を呼び出す（既存関数を流用）

### 実装タスク
1. `app.html` の `#resultPanel` セクションを Read して現在の構造を把握
2. 既存のボタングループ div の直下に新規ボタンを追加
3. CSS: `.result-button-group` クラスを定義し、`flex flex-col md:flex-row gap-2` を指定
4. `resetSession()` 関数（既存）が正しく動作するか確認

### DoD
- ✅ 結果パネルにボタンが表示される
- ✅ ボタンクリック時にミッション選択画面に戻る
- ✅ 前回のテキスト・入力欄がクリアされる
- ✅ PC で横並び、モバイルで縦積みになる
- ✅ ボタンのタッチターゲット: 56px × 56px 以上

## 3️⃣ P2: タイピング完了時のセレブレーション演出（エンゲージメント向上）

### 目的
テキスト入力完了時に「完了！」というテキスト表示のみ。ユーザーの達成感が薄い。視覚的・聴覚的な演出を強化し、リトライ意欲 / 継続利用の向上を図る。

### 仕様
1. **背景フラッシュアニメーション**
   - 画面全体が1回フラッシュ（エメラルドグリーン #10b981、不透明度 20%）
   - 継続時間: 0.4 秒

2. **パーティクル効果**
   - 星型・円形が画面上部からランダムに落下（6個）
   - 色: シアン #06b6d4、紫 #a78bfa、黄色 #fbbf24 のランダムミックス
   - アニメーション継続: 1.0 秒
   - タイミング: 完了時に 0.1 秒ディレイで開始

3. **テキスト表示**
   - 「🎉 完了！」テキストが画面中央やや上から落下
   - フェード イン + スケール（0.5 → 1.2 → 1.0）アニメーション
   - フォントサイズ: `3rem`（PC）、`2rem`（モバイル）
   - 継続時間: 1.2 秒

4. **効果音（オプション）**
   - 既存の SoundManager（`js/audio-manager.js`）を活用
   - 小さな「ポップ」音を再生（Volume: 0.3）
   - 音が無い場合、黙音でも OK（ビジュアル演出だけで十分）

### 実装タスク
1. `app.html` に CSS キーフレーム追加
   - `@keyframes celebrationFlash`
   - `@keyframes particlesFall`
   - `@keyframes textFadeInScale`

2. `app.html` に HTML要素追加
   - `<div id="celebrationEffects" class="fixed inset-0 pointer-events-none"></div>` を body 直下に配置

3. `js/features.js`（`finishText()` 関数）に演出トリガー追加
   - 完了時に `triggerCelebration()` 関数を呼び出す

4. `js/app.js` または別ファイルに `triggerCelebration()` 関数を実装
   - パーティクル生成・アニメーション実行ロジック

### DoD
- ✅ テキスト完了時に視覚演出が再生される
- ✅ 演出がPC・モバイルで同品質で表示される
- ✅ 演出終了後、結果パネルが正しく表示される
- ✅ 演出中に入力欄がフォーカスロストしない
- ✅ 複数セッション実行時に演出が毎回再生される（キャッシュ問題なし）

## 4️⃣ P2: モバイルキーボード対応 - タイピング入力欄の自動スクロール改善

### 目的
モバイル（特に iPhone Safari）で、ユーザーがタイピング入力欄をタップするとソフトキーボードが出現し、入力欄が画面外に隠れる。「どこに入力してるのか分からない」というストレス。

### 仕様
1. **入力欄（`#typingInput`、`#timeAttackInput`）フォーカス時に自動スクロール**
   - visualViewport API を使用（または scrollIntoView で代替）
   - フォーカス時に入力欄が画面上部に 40px margin を保つ位置にスクロール

2. **キーボード出現時のレイアウト調整**
   - #textDisplay（タイピング対象テキスト）のmax-heightを制限
   - モバイル時: max-height: 150px（キーボード出現後も結果が見える）

3. **視覚フィードバック**
   - 入力欄の border を薄いシアン色に（フォーカス時も既に設定済みの可能性）
   - フォーカス時に box-shadow: 0 0 10px rgba(6, 182, 212, 0.5) を追加（より強調）

### 実装タスク
1. `app.html` の `typingInput`/`timeAttackInput` に `onFocus` ハンドラを追加
   ```javascript
   onFocus="handleInputFocus()"
   ```

2. `js/app.js` に `handleInputFocus()` 関数を実装
   - visualViewport API で キーボード高さを取得
   - document.documentElement.scrollTop を調整
   - または `element.scrollIntoView({ behavior: 'smooth', block: 'center' })`

3. CSS の `#textDisplay` に `max-height: 150px` 追加（モバイルメディアクエリ内）

4. `#typingInput::focus` / `#timeAttackInput::focus` の box-shadow を強化

### DoD
- ✅ モバイルでタップして入力欄がフォーカスされた時、入力欄が画面内に留まる
- ✅ タイピング対象テキストが入力欄上部に表示される（隠れない）
- ✅ キーボード表示後も「WPM」「正確率」等の統計が見える
- ✅ PC では動作に変化なし

## 5️⃣ P3: 苦手キー練習の直リンク化 - URLクエリパラメータサポート

### 目的
ユーザーが「弱点キー [A, I, O] の練習」をしたい時、現在は統計画面で「集中練習」ボタンをクリックする必要がある。**それを URL クエリパラメータ（例: `app.html?mode=weak&keys=a,i,o`）で直接起動可能にする**と、SNS シェア / QR コード配置 / メール配信で即座に練習を始められ、リテンション向上。

### 仕様
1. **URL パラメータ対応**
   - `app.html?mode=weak&keys=a,i,o` → 苦手キー [A, I, O] の集中練習が自動起動
   - `app.html?mode=basic` → 基礎訓練が自動起動
   - `app.html?mode=code` → コード練習が自動起動
   - `app.html?difficulty=hard` → 難易度「困難」ミッションが自動起動

2. **URL パラメータのパース**
   - ページロード時に `new URLSearchParams(window.location.search)` でパラメータ取得
   - 該当するミッション/難易度に自動遷移

3. **フォールバック**
   - 無効なパラメータの場合はミッション選択画面に戻す
   - トースト通知: 「🎯 苦手キー A, I, O の練習を開始します」

### 実装タスク
1. `js/app.js` の初期化セクションに URLSearchParams パーサーを追加
2. `startMission()`/`startWeakKeyPractice()` を拡張して、パラメータ値を受け取る
3. localStorage に URL パラメータをログ（何経由で起動したか追跡）

### DoD
- ✅ `app.html?mode=weak&keys=a,i,o` で自動起動する
- ✅ 無効なパラメータ時は安全にフォールバックする
- ✅ トースト通知でユーザーに意図が伝わる

## 6️⃣ P3: キーボードショートカットのオンスクリーン表示・チュートリアル

### 目的
ゲーム内でショートカット（例: Space=一時停止、R=リセット等）が用意されていても、ユーザーは知らない。画面に「小さくショートカット表示」があれば、発見率・利用率が高まる。

### 仕様
1. **ショートカットチート シートを画面右下に小さく配置**
   - 表示内容:
     ```
     ⌨️ ショートカット
     Space: 一時停止 / 再開
     R: リセット
     ? または /: このメニュー
     ```
   - バージョン 1.7.0 で最初は「？」ボタンで開くモーダルとして実装
   - 将来的に右下固定パネルに変更可能

2. **モバイルは非表示**
   - display: none（モバイルではショートカットが存在しないため）

3. **アクセシビリティ**
   - kbd 要素を使用（HTML semantics）
   - aria-label="キーボードショートカット" を追加

### 実装タスク
1. `app.html` に `#shortcutsButton` ボタンと `#shortcutsModal` モーダルを追加
2. `js/app.js` に `toggleShortcutsModal()` 関数を実装
3. `css/app.css` に `.shortcuts-modal` スタイル定義

### DoD
- ✅ モーダルがショートカット一覧を表示する
- ✅ PC でのみ「？」ボタンが表示される
- ✅ モーダル外をクリックで閉じられる

## 7️⃣ P3: ページ離脱時のセッション保存・復元機能

### 目的
ユーザーが練習途中でブラウザタブを閉じた場合、練習データ（残り時間、入力済みテキスト、WPM等）が失われる。復元機能があれば「再開したい」ユーザーの継続率向上。

### 仕様
1. **練習中のセッション自動保存**
   - 現在の状態（time_left, typed_text, start_time, mission_name 等）を localStorage に save_session キーで保存
   - 保存頻度: 500ms ごと（CPU負荷を考慮）

2. **ページロード時に復元ポップアップ表示**
   - 「前回中断された練習を再開しますか？」モーダル
   - 「再開」ボタン → セッション復元
   - 「新しく開始」ボタン → セッション削除、ミッション選択へ

3. **セッション有効期限**
   - セッション保存から 24 時間以上経過したら自動削除（新しい練習を促す）

### 実装タスク
1. `js/app.js` の Session Manager に セッション保存・復元ロジック追加
2. ページロード時（`DOMContentLoaded` イベント）に復元確認モーダルを表示
3. localStorage キー: `cosmicTyping_savedSession`

### DoD
- ✅ セッション中断後、再訪問時に復元ポップアップが表示される
- ✅ 「再開」で前回の状態から練習が続行される
- ✅ 24 時間以上経過したセッションは表示されない
